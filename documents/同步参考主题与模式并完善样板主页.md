## 目标综述
- 将首页英雄区文案统一为“AI Application Scaffold / AI应用脚手架工程”。
- 将“光/暗”切换改为“浅色/深色”按钮，控制明暗模式；主题A/B仅控制配色（蓝色系/紫色系）。
- 明暗模式应全局生效（包括英雄区），两套主题在浅色/深色下均有对应配色。
- 为卡片增加气泡徽标，演示不同模式与主题下的视觉效果。
- 所有服务操作通过脚本执行（client-manager.sh 等）。

## 技术方案
- 主题与模式解耦：保持 `useTheme` 中的 `theme(light|dark)` 与 `palette(blue|purple)` 两个独立状态；按钮分别调用 `setTheme('light'|'dark')` 与 `setPalette('blue'|'purple')`。
- 变量化与双态主题：在 `src/index.css` 中为每个主题分别声明浅色与深色变量集，例如：
  - `:root[data-theme="blue"]{ --bg-light/...; --surface-light/...; --accent-light/... }`
  - `:root.dark[data-theme="blue"]{ --bg-dark/...; --surface-dark/...; --accent-dark/... }`
  - 通过派生变量 `--bg/--surface/--accent` 指向当前模式的对应值，以统一组件使用。
- 英雄区切换：为英雄区新增 `--hero-bg`（浅色/深色）与依赖 `--accent` 的渐变，`.hero{ background: radial-gradient(...var(--accent)), var(--hero-bg) }`，从而在浅色与深色下均切换背景。
- 气泡徽标：新增 `.bubble` 与 `.bubble-alt` 两种视觉（圆角、描边与透明度由主题变量控制），在卡片标题右侧或内容内展示“信息1”、“信息2”等示例。
- Tailwind 配置：`tailwind.config.ts` 中将 `portal/alt` 改为 `blue/purple`（或建立别名映射），并补充浅色/深色核心色值以便 CSS 变量引用。
